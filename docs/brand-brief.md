# DoReMingo — brand brief

Source of truth for brand/identity work. Pull from this when prompting
claude.ai/design (or any designer). Everything below is the de-facto brand
already living in the product — extend it, don't replace it.

## What it is
A free, cheerful ear-training game (web + iOS). Teaches you to recognise
musical notes by ear via functional ear training (Kodály movable-do). No
musical education needed. Works on phone and desktop.

## Personality
Warm, playful, encouraging, unintimidating. **No fail states** — wrong answers
teach as much as right ones. Anti-"serious music theory". A happy flamingo, not
a metronome and a frown.

Voice: friendly, second-person, light. "Hear a note. Feel where it lives."
Avoid: jargon, pitch names, anything that implies you need to be a musician.

## Mascot
A flamingo 🦩 (the "-mingo" in DoReMingo). Currently only an emoji. Needs a real
character: cheerful, rounded, expressive (it reacts — happy / party / sad).
Should work tiny (app icon, favicon) and large (hero).

## Name
**DoReMingo** — pun on Do-Re-Mi (solfège) + flamingo. Keep the play on solfège
syllables (Do, Re, Mi, Fa, Sol, La, Ti) available as a motif.

Background uses soft radial glows (pink top-left #ffe3ee, blue bottom-right
#dff6ff) over the cream.

## Type
Rounded, friendly sans. Currently `ui-rounded` (SF Rounded). Pick a brand
typeface in that spirit (geometric/rounded, high warmth) with a free/web-licensed
option for shipping.

## Shape language
Pills and big rounded buttons (border-radius 999px), drop-shadows as "physical"
button depth, confetti on wins. Soft, tactile, toy-like. No hard corners.

## Deliverables wanted from Claude Design
1. Logo (wordmark + flamingo mark; horizontal + stacked; mono fallback).
2. App icon / favicon that reads at 16px and 1024px.
3. Refined colour system (the palette above, formalised: roles, light/dark,
   accessible text contrast pairs).
4. Type scale + pairing.
5. Mascot character sheet with a few expressions (idle/happy/party/sad).
6. One-page styleguide: do/don't, spacing, button states, voice examples.

## Hard constraints
- Must stay accessible (AA contrast for text).
- Must scale down to an app icon and a 16px favicon.
- Keep it free/low-cost to ship (no expensive font/icon licences).
- Don't drift "corporate" — it's a toy, keep it joyful.
